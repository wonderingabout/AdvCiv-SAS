Beginning of quote:
"
Apparently,
TXT_KEY_PEDIA_CATEGORY_CONCEPT_NEW
is the name of the "BtS Concepts" category. So that can be changed to "Mods information" in CIV4GameText_advc_pedia.xml or can be moved to a different game text file and changed there. (The original definition is in CIV4GameText_Objects_BTS.xml.) The "Game Concepts" text key is TXT_KEY_PEDIA_CATEGORY_CONCEPT in CIV4GameTextInfos_Objects.xml (base game); can be re-defined to "BtS Concepts". Regarding the placement of concepts in categories, Sevopedia has these functions:
Spoiler : 
Python:
	def getNewConceptInfo(self, id):
		info = gc.getNewConceptInfo(id)
		if not self.isShortcutInfo(info) and not self.isTraitInfo(info):
			return info
		return None

	def isTraitInfo(self, info):
		return info.getType().find("_TRAIT_") != -1

	def isShortcutInfo(self, info):
		return info.getType().find("SHORTCUTS") != -1
So, if it has "_TRAIT_" or "SHORTCUTS" in its type string, it's not a BtS Concept. Everything else in BasicInfos\Civ4NewConceptInfos.xml is a BtS Concept. The pre-BtS Concepts are taken from CIV4BasicInfos.xml (the BtS version of that file). Those ConceptInfo or NewConceptInfo elements specify one text key each for the heading and for the exposition:
XML:
<ConceptInfo>
	<Type>CONCEPT_ANIMALS</Type>
	<Description>TXT_KEY_CONCEPT_ANIMALS</Description>
	<Civilopedia>TXT_KEY_CONCEPT_ANIMALS_PEDIA</Civilopedia>
</ConceptInfo>
So, to move the BtS Concepts into Game Concepts, one could copy Civ4BasicInfos.xml from BtS into the mod, cut-and-paste all the non-trait, non-shortcut NewConceptInfo elements from Civ4NewConceptInfos.xml into CIV4BasicInfos.xml and change them from NewConceptInfo to ConceptInfo. And then create NewConceptInfo elements in Civ4NewConceptInfos.xml for the content you want in there, e.g.
XML:
		<NewConceptInfo>
			<Type>CONCEPT_KMOD_RULES</Type>
			<Description>TXT_KEY_KMOD_RULES</Description>
			<Civilopedia>TXT_KEY_KMOD_RULES_PEDIA</Civilopedia>
		</NewConceptInfo>
(Could be named anything, I think; just so long as it doesn't say "trait" or "shortcut".) Plus the corresponding text key definitions in a game text file. All just in theory. Might even work if Sevopedia is disabled via the BUG menu. Ignoring that possibility, there are probably more surgical ways to achieve what you want through changes in SevoPediaMain.py. But you'd need to change a bunch of (small) functions there, not just the three I included above.
"
